## 1. סקירה כללית

מערכת אוטומציה מלאה לניהול KPI שבועי ובונוסים חודשיים עבור מוקד תיאום פגישות (מיקור חוץ), לקוח וולטה סולאר.

**דרישות עיקריות:**
- פועלת מהדפדפן בלבד (ללא התקנת תוכנה מקומית)
- קולטת קבצי Excel קיימים ומחשבת אוטומטית
- מבקשת אישור מהמשתמש לפני שליחת מייל
- תומכת בניהול דינמי של נציגים
- מייצאת ל-Excel
- שולחת מיילים HTML מקצועיים

**פלטפורמה:** Streamlit Cloud (Python, ענן, חינמי, ללא התקנה)

## 2. מקורות נתונים

### 2.1 קובץ נוכחות (`נוכחות לפי מחלקה.xlsx`)
- גיליון: `וולטה סולאר`
- שורה אחת לעובד לכל יום
- עמודות רלוונטיות: `תאריך`, `מספר עובד`, `שם עובד`, `סה"כ כללי`
- **חישוב שעות:** `sum(סה"כ כללי per day) − 1 שעה × מספר ימי עבודה`
  - ההפחתה היא תמיד שעה אחת קבועה ליום, ללא תלות במשך המשמרת

### 2.2 קובץ Voicenter (`Calls & employment performance statistics.xls`)
- פורמט: HTML בקידוד UTF-16 מוסווה כ-XLS
- קריאה: `pd.read_html(file, encoding='utf-16')`
- עמודות: `משתמש`, `סה"כ שיחות`, `נענו`, `שיחות שלא נענו`, `אחוז תעסוקה נטו`, `סה"כ שיחות יוצאות`

### 2.3 קלט ידני (מסך הזנה)
לכל נציג:
- מספר **תיאומים** (פגישות שתואמו)
- מספר **פניקס** (עסקאות שנסגרו בשיחה ראשונית)
- מספר **שיחות סרק**

### 2.4 קובץ משובים (`משובים וולטה חודש X.xlsx`)
- גיליון נפרד לכל נציג
- שורה `ציון משוב` בראש הגיליון — הציון הסופי (סקאלה 0–10)
- משמש רק לחישוב בונוס חודשי

## 3. חישובי KPI

| מדד | נוסחה | מקור |
|---|---|---|
| שעות עבודה | `Σ(סה"כ כללי) − (מספר ימים × 1h)` | נוכחות |
| פגישות לשעה (נציג) | `תיאומים ÷ שעות עבודה` | מחושב |
| פגישות לשעה (מוקד) | `Σתיאומים ÷ Σשעות (כל נציגים פעילים)` | מחושב |
| אחוז תעסוקה | מתוך Voicenter | Voicenter |
| אחוז סרק | `שיחות סרק ÷ שיחות שנענו` | מחושב |
| ציון משוב | שורת `ציון משוב` מגיליון נציג | קובץ משובים |

**נציג ללא שעות (0 ימי עבודה):** מסומן "לא פעיל" ומוחרג מחישוב קצב המוקד.

## 4. מודל בונוס חודשי

### 4.1 בונוס נציגים

#### תיאומים (בסיס + בונוס מוקד)
| תנאי | בונוס |
|---|---|
| נציג ≥ 1.0 פגישה/שעה | 5₪ × מספר תיאומים |
| נציג < 1.0 פגישה/שעה | 4₪ × מספר תיאומים |
| **מוקד כולו ≥ 1.0 פגישה/שעה** | **+1₪ × מספר תיאומים (לכל נציג)** |

#### תעסוקה
| אחוז תעסוקה | בונוס |
|---|---|
| ≥ 35% | 300₪ |
| ≥ 30% | 200₪ |
| < 30% | 0₪ |

#### שיחות סרק
| אחוז סרק | בונוס |
|---|---|
| ≤ 2% | 150₪ |
| ≤ 3% | 100₪ |
| ≥ 4% | 0₪ |

#### ציון משוב (סקאלה 0–10)
| ציון | בונוס |
|---|---|
| ≥ 8.5 | 150₪ |
| ≥ 8.0 | 100₪ |
| < 8.0 | 0₪ |

#### פניקס
| | |
|---|---|
| לנציג | 50₪ × מספר עסקאות |
| חיוב ללקוח | 100₪ × מספר עסקאות |

### 4.2 בונוס מנהל (לפי קצב מוקד)
| קצב מוקד | בונוס |
|---|---|
| ≥ 1.0 פגישה/שעה | 2,000₪ |
| ≥ 0.8 פגישה/שעה | 1,600₪ |
| < 0.8 פגישה/שעה | 1,200₪ |

**כל הספים מוגדרים בקובץ config ועריכים מ"הגדרות" — אף מספר לא קבוע בקוד.**

## 5. ממשק המשתמש (5 מסכים)

### מסך 1 — Dashboard
- טבלת ביצועים שבועית/חודשית לכל הנציגים
- גרף מגמות פגישות/שעה
- אינדיקטור ירוק/אדום לעמידה ביעד המוקד

### מסך 2 — KPI שבועי
1. העלאת קובץ נוכחות + קובץ Voicenter
2. טופס קלט ידני: תיאומים / פניקס / סרק לכל נציג
3. תצוגה מקדימה של דוח + כפתור "ייצוא Excel"
4. תצוגה מקדימה של 2 מיילים → אישור → שליחה
   - מייל להנהלה: טבלת KPI כוללת
   - מייל לנציגים: ביצועים אישיים בלבד

### מסך 3 — בונוסים חודשיים
1. העלאת קובץ נוכחות + Voicenter + משובים
2. קלט ידני: תיאומים / פניקס / סרק חודשיים
3. תצוגה מקדימה של חישוב בונוסים + "ייצוא Excel" (3 גיליונות)
4. תצוגה מקדימה של 3 מיילים → אישור → שליחה:
   - **ללקוח (וולטה):** שעות × תעריף + פניקס × 100₪ + Excel מצורף
   - **להנהלה:** סיכום כולל עם בונוס מנהל
   - **לכל נציג:** פירוט אישי בלבד

### מסך 4 — ניהול נציגים
- הוספה / הסרה / עריכת נציגים
- שדות: שם, מספר עובד, כתובת מייל, סטטוס (פעיל/לא פעיל)
- נציג לא פעיל לא נכלל בחישובים

### מסך 5 — הגדרות
- עריכת ספי בונוס (תיאומים, תעסוקה, סרק, משוב, מנהל)
- תעריפי פניקס (נציג / ללקוח)
- כתובות מייל של נמענים קבועים (הנהלה, לקוח)
- פרטי SMTP לשליחת מיילים

## 6. ייצוא Excel

3 גיליונות בחישוב חודשי:

| גיליון | תוכן |
|---|---|
| `לתשלום` | שם, מספר עובד, בונוס סופי |
| `מפרוט` | פירוט כל רכיב בונוס לכל נציג |
| `חיוב ללקוח` | שעות × תעריף + פניקס × 100₪ |

גיבוי: לפני כל ייצוא, הגרסה הקודמת נשמרת עם חותמת תאריך.

## 8. טיפול בשגיאות ומקרי קצה

| מצב | טיפול |
|---|---|
| שם נציג שונה בין קבצים | התאמה לפי `מספר עובד`, לא לפי שם |
| נציג ב-Voicenter אך חסר בנוכחות | התראה ברורה + המשך |
| קובץ משובים חסר לנציג | ציון = null, בונוס משוב = 0 + הסבר |
| נציג עם 0 שעות | מסומן "לא פעיל", מוחרג מקצב מוקד |
| Voicenter בפורמט שגוי | הודעת שגיאה + הנחיה לגבי הפורמט הנכון |
| שליחת מייל נכשלת | הודעה ברורה + אפשרות לנסות שוב |

**אישור לפני שליחה:** כפתור "שלח" מופיע רק לאחר סימון "בדקתי ואישרתי את התצוגה המקדימה".

## 10. היסטוריה וניתוח נתונים

### שמירת לוג חודשי
בסיום כל חישוב חודשי, המשתמש יכול לשמור snapshot של החודש לקובץ `data/history.json`.

**מה נשמר לכל חודש:**
```json
{
  "month": "2026-06",
  "label": "יוני 2026",
  "saved_at": "2026-07-01T10:00:00",
  "center_rate": 1.02,
  "center_met_target": true,
  "manager_bonus": 2000,
  "total_billing": 45600,
  "agents": [
    {
      "name": "טום סורסקי",
      "hours": 113.0,
      "meetings": 142,
      "meetings_per_hour": 1.257,
      "occupancy_pct": 0.36,
      "idle_pct": 0.005,
      "feedback_score": 8.52,
      "phoenix": 5,
      "bonus_total": 1380
    }
  ]
}
```

### מסך היסטוריה — ניתוחים שיוצגו

| ניתוח | תיאור |
|---|---|
| קצב מוקד לאורך זמן | גרף קו — center_rate לפי חודש |
| תיאומים לשעה — כל נציג | גרף עמודות מקובצות לפי חודש |
| מגמת בונוסים | סה"כ בונוס לנציג לפי חודש |
| ביצוע מול יעד | % חודשים שהמוקד עמד ביעד |
| מגמת פניקס | ספירת עסקאות לפי חודש |
| ניתוח חיוב ללקוח | סה"כ חיוב לפי חודש |

### גיבוי
- כפתור "הורד היסטוריה (JSON)" לגיבוי ידני
- כפתור "העלה היסטוריה" לשחזור אחרי redeploy
